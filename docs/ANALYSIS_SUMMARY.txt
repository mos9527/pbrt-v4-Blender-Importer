================================================================================
KEY FINDING
================================================================================

The horizontal flipping is NOT caused by a bug in the coordinate system 
conversion. The 90-degree X-axis rotation is CORRECT for converting from 
PBRT's Y-up to Blender's Z-up.

Most flipping issues are caused by:
1. Negative scale values in the .pbrt source file (intentional mirrors)
2. PLY files generated in a different coordinate system
3. Direct Transform matrices with negative determinants

================================================================================
TOP CAUSES OF FLIPPING
================================================================================

1. PBRT file has Scale -X directive (CRITICAL)
   - Detection: grep "Scale -" *.pbrt
   - This is intentional mirroring in the source

2. PLY file coordinate system mismatch (CRITICAL)
   - PLY in Z-up but PBRT expects Y-up
   - Detection: Open PLY in tool that shows axes
   - Solution: Re-export PLY in Y-up coordinates

3. Negative Transform matrix (CRITICAL)
   - Direct Transform with -1 on diagonal
   - Detection: grep "Transform.*-" *.pbrt

4. Negative global_scale (HIGH)
   - Parameter not validated in code
   - Detection: Check import dialog

5. Include file with Scale -1 (HIGH)
   - Recursive issue in included files
   - Detection: Check all included files

6. ObjectInstance with negative scale (HIGH)
   - Instance matrix has negative determinant
   - Detection: Inspect instance transforms

================================================================================
VERIFICATION
================================================================================

The 90-degree X-rotation is CORRECT:
  - Determinant = +1.0 (no reflection)
  - Handedness preserved (right-handed)
  - Necessary for Y-up to Z-up conversion
  - Matches PBRT documentation

DO NOT try to "fix" this rotation!

================================================================================
COORDINATE SYSTEM TRANSFORMATION
================================================================================

PBRT (Y-up, Z-forward):          Blender (Z-up, Y-forward):
      Y                                   Z
      |                                   |
      | Z                                 | Y
      |/                                  |/
  ----+----- X                        ----+----- X

Conversion: Rotate +90 degrees around X-axis
Effect: (x, y, z) in pbrt becomes (x, -z, y) in Blender

================================================================================
TRANSFORMATION PIPELINE
================================================================================

1. Parser builds CTM (Current Transform Matrix)
   - Accumulates all transforms in pbrt file
   - Stored as 16 floats in column-major format
   - May contain negative values (mirrors)

2. Builder creates correction matrix
   correction = Rotation(90,X) @ Scale(global_scale,4)

3. Builder applies to all objects
   world = correction @ pbrt_world_matrix

4. Result in Blender Z-up coordinates
   - All transforms preserved (including mirrors)
   - Camera view direction corrected with _CAM_FLIP

================================================================================
DEBUGGING STEPS (IN ORDER)
================================================================================

Step 1: Search for negative scales
  grep -r "Scale.*-[0-9]" .
  (These are intentional mirrors in the source)

Step 2: Verify PLY coordinate system
  - Open each PLY file in a tool showing axes
  - Verify it's in Y-up (not Z-up)
  - If Z-up, re-export in Y-up coordinates

Step 3: Check for negative Transform matrices
  grep -r "Transform" . | grep "\-"

Step 4: Verify global_scale is positive
  - Check import dialog
  - Must be > 0

Step 5: Check Include files
  - Find all Include directives
  - Check each included file for Scale -1
